Help & Support

Welcome to the official Help & Support page of TakeGo. TakeGo is a token-based takeaway order management system developed as part of a Software Engineering academic project. This section provides complete guidance for Customers, Stall Owners, and academic evaluators regarding the platform’s usage and functionality.

Frequently Asked Questions (FAQs)

1. How do I register on TakeGo?

To use the system, you must first register yourself based on your role:

  • Customer: Register as a Customer to browse stalls and generate order tokens.
  • Stall Owner: Register as a Stall Owner to manage products and incoming orders.

After registration, log in using your credentials to access your respective dashboard.

2. How do Customers place an order?

  • Register and log in as a Customer.
  • Browse the list of available stalls.
  • Select products from a stall.
  • Choose the quantity (subject to availability).
  • Click on “Generate Token” to confirm the order.

Upon successful order placement, a unique token number is generated and displayed. The system also reduces product availability automatically.

3. How does the Token System work?

Each confirmed order is assigned a sequential token number per stall. The token system simulates queue management within a takeaway environment. Tokens help demonstrate order tracking, preparation sequencing, and stall-level order management.

4. How is preparation time calculated?

The estimated preparation time is calculated dynamically based on:

  • The preparation time defined for each product.
  • The quantity ordered.
  • The queue logic implemented in the system.

This feature is implemented purely for academic demonstration of backend logic and database aggregation.

5. How can Customers track their orders?

Customers can access the "My Orders" section after logging in. This section displays:

  • Token number
  • Order status (Pending, Accepted, Ready, Cancelled, Rejected)
  • Total price
  • Order items and quantities
  • Estimated preparation time

6. How do Stall Owners manage orders?

  • Register and log in as a Stall Owner.
  • Access the Owner Panel dashboard.
  • Add, edit, or delete products.
  • View incoming customer orders.
  • Update order status to Accepted, Ready, Cancelled, or Rejected.

Stall Owners can also manage product availability, which directly impacts token generation.

7. What happens if a product is deleted?

If a product is removed by a Stall Owner, previously generated orders remain visible in the system. Deleted products are displayed as “[Deleted Product]” in order history for data consistency and audit demonstration.

8. Does TakeGo support real payments?

No. TakeGo does not process real financial transactions. All pricing, order totals, and token generation features are simulated for academic evaluation purposes only.

Technical Information

  • Developed using Flask (Python)
  • SQLite Database integration
  • Role-based authentication (Customer & Stall Owner)
  • Password hashing for security demonstration
  • Token-based order queue logic
  • AJAX-based token generation without full page reload
  • Flash message system for user feedback

Common Issues & Troubleshooting

Login Required Error

If you attempt to generate a token without logging in, the system will prevent order creation and display a login notification.

Product Out of Stock

Token generation will be blocked if product availability is insufficient.

Session Expiry

If your session expires, you will be required to log in again to continue using the platform.

Academic Disclaimer

TakeGo is developed strictly as an academic Software Engineering project. It is not a commercial product and is not intended for real-world deployment. All data, tokens, and order flows exist solely for educational and demonstration purposes.

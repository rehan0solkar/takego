1. Nature of the Platform

TakeGo is developed strictly for academic, demonstration, and evaluation purposes. It is not a commercial product and is not intended for real-world business operations. No real financial transactions are processed through this system.

2. User Roles and Responsibilities

The platform supports two primary roles:

  • Customers: Users who register, browse stalls, generate order tokens, and track order status.
  • Stall Owners: Users who manage products, update availability, accept/reject orders, and mark orders as ready.

Users are responsible for using their assigned role appropriately. Unauthorized access or misuse of role-based functionality is prohibited.

3. Account Registration and Security

  • Users must register before accessing system features.
  • Usernames must follow platform validation rules.
  • Passwords are stored using hashing techniques for security demonstration.
  • Users are advised not to use real-world sensitive credentials.

While security mechanisms are implemented for educational purposes, this system should not be treated as a production-grade secure application.

4. Token-Based Order System

When a customer places an order, the system generates a unique token number. Tokens are generated sequentially per stall to simulate queue management logic.

The token mechanism is intended to demonstrate:

  • Database transactions
  • Concurrent order handling logic
  • Queue simulation per stall
  • Status updates (Pending, Accepted, Ready, Rejected, Cancelled)

5. Orders and Availability

  • Orders can only be placed if product availability is sufficient.
  • Availability is automatically reduced upon successful order placement.
  • Stall owners may update product availability at any time.
  • Order records may be soft-deleted for system demonstration purposes.

Order preparation time displayed is calculated based on product prep-time logic and is intended only for simulation.

6. No Payment Processing

TakeGo does not integrate any real payment gateway. All pricing displayed is for academic demonstration only. No real financial processing occurs within this platform.

7. Data Handling and Storage

The system uses a SQLite database for demonstration of relational data management. Data stored may include:

  • Username
  • User role
  • Product information
  • Order records
  • Generated tokens

Data may be reset, modified, or cleared at any time during development, testing, or academic evaluation.

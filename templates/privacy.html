Privacy Policy

Last Updated: February 2026

This Privacy Policy explains how TakeGo collects, stores, and processes information when users interact with the platform. TakeGo is a token-based takeaway order management system developed as part of a Software Engineering academic project. The system is created strictly for educational demonstration and evaluation purposes.

1. Nature of This Project

TakeGo is not a commercial application and is not connected to any real-world food delivery or payment processing service. All transactions, tokens, and order flows are simulated for academic demonstration only. No real financial transactions are processed within this system.

2. Information Collected

The platform may collect and store the following information:

  • Username provided during registration
  • User role (Customer or Stall Owner)
  • Hashed password for authentication (not stored in plain text)
  • Product and stall information added by Stall Owners
  • Order details (product ID, quantity, price calculation)
  • System-generated token numbers
  • Order timestamps (creation time, acceptance time)
  • Order status updates (pending, accepted, ready, cancelled, rejected)
  • Logical deletion flags for soft-deleted orders

3. Purpose of Data Collection

  • To enable secure user authentication
  • To support role-based access control (Customer / Owner)
  • To simulate token-based takeaway order generation
  • To allow stall owners to manage product inventory and orders
  • To demonstrate database relationships (orders, order_items, products)
  • To calculate estimated preparation time
  • To evaluate system functionality during academic assessment

4. Authentication and Security

Passwords are stored using hashing mechanisms to demonstrate secure development practices. The system uses role-based validation to restrict access to customer and owner dashboards.

Although security best practices are demonstrated, this project is not intended for production use and should not be used to store real sensitive information.

5. Order and Token System

Tokens are automatically generated by the system using incremental logic per stall. Token numbers are used to simulate queue management in a takeaway environment.

Orders move through different statuses such as:

  • Pending
  • Accepted
  • Ready
  • Cancelled
  • Rejected

Stall owners can update order status through the Owner Panel. Customers can track their order status from the order history section.

6. Soft Deletion of Orders

The system uses logical deletion (soft delete) for orders. Instead of permanently removing records from the database, an "is_deleted" flag may be applied for demonstration purposes. This helps illustrate database integrity and historical record retention.

7. Data Storage

All data is stored in a SQLite database used strictly for development and academic demonstration. The database may run locally or on a development server.

The system does not collect:

  • Credit card information
  • Banking details
  • Government identification numbers
  • Personal contact information beyond username

8. Data Sharing

TakeGo does not sell, distribute, or share user data with third parties. All information remains within the academic project environment.

9. User Responsibility

Since this is an academic project, users are advised not to register using real sensitive or confidential data.

10. Changes to This Policy

This Privacy Policy may be updated if the academic project requirements change. Updates will be reflected with a revised “Last Updated” date.
